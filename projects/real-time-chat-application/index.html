Problem

What needed to improve

Messaging systems are only useful when they feel immediate and dependable. The core challenge was to reduce chat latency, keep status updates current, and make message retrieval fast enough that the application remained responsive during active use.

Approach

How I built it

I designed the application around Angular on the client, Spring Boot on the backend, WebSocket for real-time communication, and MySQL for persistence. The work focused on coordination between frontend behavior and backend delivery so live updates and message history felt fast together.

Outcome

What changed

The application delivered lower latency and faster retrieval while supporting the real-time expectations users bring to chat. It highlights the full-stack coordination required for responsive systems where interface timing and backend behavior must stay aligned.

Highlights

Main points

  • Optimized for real-time responsiveness rather than treating messaging as a standard request-response flow.
  • Kept presence, status updates, and stored message history working together as one experience.
  • Used WebSocket where live interaction mattered most instead of forcing everything through slower patterns.
  • Focused on lower latency and retrieval speed because those changes are directly visible to users.
