Problem

What needed to improve

Legacy analysis work depended heavily on spreadsheets, which slowed teams down, increased manual effort, and made the workflow harder to scale or onboard people into. The goal was not just to rebuild the interface, but to move the process into a more dependable application model.

Approach

How I built it

I helped modernize the workflow with an Angular frontend and ASP.NET Core APIs, translating the existing process into a cleaner browser-based experience. The work emphasized usability, structured data flow, and steady stakeholder communication through regular demos and progress updates.

Outcome

What changed

The new platform reduced manual steps, improved responsiveness, and gave users a clearer path through the analysis process. It also turned a fragile, spreadsheet-heavy workflow into a product surface that was easier to maintain and easier for teams to adopt.

Highlights

Main points

  • Modernized a real workflow instead of simply redesigning the interface around the same manual process.
  • Reduced onboarding friction by making the application easier to understand than the prior spreadsheet approach.
  • Used stakeholder feedback to guide the implementation and keep the work aligned with real operational needs.
  • Focused on reliability and clarity so the tool could support day-to-day use rather than one-off reporting.
