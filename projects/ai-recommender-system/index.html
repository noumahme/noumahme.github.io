Problem

What needed to improve

The recommendation flow needed to feel smarter without becoming slow, noisy, or disconnected from product needs. That meant combining language-model behavior with structured data in a way that improved relevance while still respecting latency and usability constraints.

Approach

How I built it

I implemented the system in Python using the OpenAI API and PostgreSQL, shaping prompt behavior and data flow around recommendation quality and performance. The work focused on practical evaluation, faster queries, and a product-oriented use of LLMs rather than treating the model as a novelty feature.

Outcome

What changed

The recommender produced stronger engagement and better recommendation quality while also improving response speed. It demonstrated the kind of AI application work I care about most: model usage that fits a real workflow, measurable outcomes, and clear engineering tradeoffs.

Highlights

Main points

  • Combined LLM behavior with structured backend data instead of relying on the model alone.
  • Treated latency as part of the product quality bar, not as an afterthought.
  • Used evaluation and prompt shaping to improve relevance in a concrete recommendation context.
  • Framed the project around user value and performance, not only around the underlying model.
